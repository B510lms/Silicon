❓ Q1：你这个项目主要做了什么？

✅ 答（模板）：

这是一个基于 Vue3 + TypeScript 的后台管理系统，主要包含商品管理和权限管理两大模块。
我主要负责整体项目搭建、登录鉴权、权限控制、商品模块开发以及请求封装等核心功能。

❓ Q2：token 是怎么做登录鉴权的？

✅ 答：

登录成功后，后端返回 token，我将 token 存储在 localStorage 中。
在 axios 请求拦截器中统一携带 token 到请求头。
同时在路由守卫中判断是否存在 token：

有 token：放行
无 token：跳转登录页

❓ Q3：token 过期怎么处理？

✅ 答（进阶一点）：

在 axios 响应拦截器中判断返回状态码：

如果是未授权（如 401），说明 token 失效
清除本地 token
跳转登录页重新登录

❓ Q4：你 axios 是怎么封装的？

✅ 答：

我主要做了三点封装：

创建 axios 实例，统一 baseURL 和 timeout
请求拦截器：统一携带 token
响应拦截器：统一处理错误（如提示 message）

❓ Q5：动态路由是怎么实现的？

✅ 答（核心）：

动态路由是基于用户角色来控制的，大致流程是：

登录后获取用户角色信息
根据角色过滤出可访问的路由
使用 router.addRoute 动态添加路由
同时生成对应菜单进行渲染

❓ Q6：路由和菜单怎么关联？

✅ 答：

菜单数据和路由结构是统一的，通过路由的 meta 信息（如 title、icon）来渲染菜单，实现一套数据驱动两套 UI。

❓ Q7：页面刷新后动态路由丢失怎么办？

✅ 答（关键加分）：

因为动态路由是运行时添加的，刷新会丢失。
解决方案是：

将用户信息和权限数据存储（如 localStorage 或 pinia）
页面刷新后重新执行动态路由加载逻辑

❓ Q8：除了路由权限，还有做按钮权限吗？

✅ 答（如果你没做，也可以这么说）：

当前项目主要实现了菜单和路由权限控制，按钮级权限可以通过 v-if 或自定义指令扩展实现。

👉 如果你想装强一点：

❓ Q9：SPU 和 SKU 你怎么理解？

✅ 答（必须会）：

SPU：标准产品单位，表示一种商品（如 iPhone 14）
SKU：库存单位，表示具体规格（如 iPhone 14 黑色 128G）

❓ Q10：商品列表分页怎么做？

✅ 答：

前端维护 page 和 pageSize，调用接口时传递参数
后端返回 total 和当前页数据
使用 Element Plus 的分页组件进行联动更新
